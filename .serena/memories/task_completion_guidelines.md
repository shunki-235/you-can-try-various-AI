# タスク完了時のチェックリスト

このプロジェクトで機能追加や修正タスクを完了とみなす際の目安です。

1. **実装が要件に沿っているか確認**
   - `llm-multi-provider-requirements.md` に記載された要件・画面設計と差分がないかを確認する。
   - 仕様を変更した場合は、README ではなく要件ドキュメント側を更新する方針。

2. **ビルド・静的解析**
   - `pnpm lint` を実行し、ESLint のエラーがないことを確認。
   - `pnpm typecheck` を実行し、型エラーがないことを確認。

3. **テスト**
   - 関連するユニットテスト (`pnpm test`) を追加・更新し、パスしていることを確認。
   - 重要フローの場合は E2E テスト (`pnpm e2e`) を追加/更新し、主要シナリオが通ることを確認。

4. **UI/UX & アクセシビリティ**
   - レイアウト崩れがないか、主要ブレークポイント（モバイル/タブレット/PC）でざっと確認。
   - キーボード操作（Tab 移動、Enter・Space など）で主要操作が行えるかを確認。

5. **セキュリティと機微情報**
   - `.env.*` に置くべき値をコードにベタ書きしていないか確認。
   - ログ出力に API Key やフルプロンプトなどの機微情報が含まれていないか確認。

6. **ドキュメント**
   - 仕様やフローに変更があれば、`llm-multi-provider-requirements.md` に反映する。
   - 実装の使い方レベルは必要に応じて `README.md` に追記する。
